RESUMEN:
El curso completo de Generative Engine Optimization (GEO) implementa la metodología F1-F6, la primera aproximación sistemática en español para optimizar contenido web para modelos de lenguaje generativo. Cada módulo (F1 a F6) aborda aspectos específicos de la citabilidad y comprensión por IA.

METODOLOGÍA F1-F6 DETALLADA:

=== MÓDULO F1: FRAGMENTACIÓN ===
Objetivo: Estructurar contenido en bloques semánticamente coherentes y citables.

Principios fundamentales:
• Párrafos de 100-150 palabras máximo
• Una idea principal por fragmento
• Conectores semánticos explícitos
• Uso de listas y enumeraciones
• Separación visual clara entre conceptos

Técnicas específicas:
1. Fragmentación por subtemas
2. Uso de headers H2-H6 descriptivos
3. Implementación de <section> y <article>
4. Bloques de código con contexto
5. Fragmentos destacados (snippets)

Métricas F1:
- Longitud promedio de párrafos
- Coherencia temática por fragmento
- Facilidad de extracción de información
- Comprensibilidad fuera de contexto

=== MÓDULO F2: JERARQUÍA SEMÁNTICA ===
Objetivo: Organizar información con estructura lógica comprensible para LLMs.

Estructura recomendada:
• H1: Título principal (único por página)
• H2: Secciones principales
• H3: Subsecciones temáticas
• H4-H6: Detalles específicos
• Uso consistente de HTML5 semántico

Elementos HTML5 clave:
- <main>: Contenido principal
- <section>: Agrupaciones temáticas
- <article>: Contenido independiente
- <aside>: Información complementaria
- <nav>: Navegación contextual

Métricas F2:
- Profundidad de anidamiento
- Consistencia de estructura
- Accesibilidad semántica
- Navegabilidad lógica

=== MÓDULO F3: AUTORIDAD GENERATIVA ===
Objetivo: Construir credibilidad específica para modelos de IA.

Factores de autoridad:
• Citaciones y referencias verificables
• Autoría clara y experticia demostrable
• Fechas de publicación y actualización
• Enlaces a fuentes primarias
• Transparencia metodológica

Elementos técnicos:
1. Meta tags de citación académica
2. Schema.org de autoría
3. Indicadores temporales
4. Badges de verificación
5. Historial de actualizaciones

Métricas F3:
- Tasa de citación por LLMs
- Precisión de atribuciones
- Frecuencia de recomendación
- Confianza percibida por IA

=== MÓDULO F4: DATOS ESTRUCTURADOS ===
Objetivo: Implementar Schema.org para comprensión automatizada.

Esquemas prioritarios:
• Article/NewsArticle
• EducationalOrganization
• Course/LearningResource
• FAQ/QAPage
• HowTo/Guide

Implementación JSON-LD:
- Metadatos de contenido
- Información de autoría
- Datos de publicación
- Relaciones entre entidades
- Contexto semántico

Métricas F4:
- Validación Schema.org
- Riqueza de metadatos
- Interconexión de entidades
- Precisión semántica

=== MÓDULO F5: OPTIMIZACIÓN TÉCNICA ===
Objetivo: Configuraciones específicas para crawlers de LLMs.

Aspectos técnicos:
• Velocidad de carga óptima
• Accesibilidad para bots
• URLs semánticamente descriptivas
• Sitemap específico para IA
• Headers HTTP apropiados

Archivos de configuración:
1. robots.txt optimizado para LLMs
2. llm.txt (directivas específicas)
3. sitemap.xml extendido
4. Meta tags avanzados
5. Canonical URLs estratégicas

Métricas F5:
- Tiempo de rastreo
- Completitud de indexación
- Eficiencia de descubrimiento
- Calidad técnica general

=== MÓDULO F6: MEDICIÓN Y ANÁLISIS ===
Objetivo: Métricas de citabilidad y rendimiento en modelos generativos.

KPIs específicos de GEO:
• Tasa de citación directa
• Precisión de parafraseado
• Frecuencia de recomendación
• Retención de contexto
• Autoridad generativa percibida

Herramientas de medición:
1. Tests directos con LLMs
2. Análisis de menciones
3. Tracking de referencias
4. Auditorías de precisión
5. Monitoreo de autoridad

Métricas F6:
- ROI de citabilidad
- Precisión informacional
- Alcance generativo
- Impacto en awareness

RUTAS DE APRENDIZAJE:
• Principiante: F1 → F2 → F6
• Intermedio: F1 → F2 → F3 → F4 → F6
• Avanzado: Secuencia completa F1-F6
• Especialista: F6 → Optimización continua
